# OpenSofa Web MVP - Simplified Product Specification

**ULTRATHINK Applied: Intentional Minimalism**

---

## 1. The Pivot

### What We Learned from Competitors

| Product | Approach | Gap |
|---------|----------|-----|
| **OpenCode** | Client/server, mobile app planned | No push notifications |
| **AgentAPI** | Has `/chat` UI built-in | No remote access, single session |
| **OpenSofa** | WhatsApp + multi-session | No web interface for terminal |

### The Key Insight

AgentAPI **already has a web chat UI** at `http://localhost:3284/chat`. We don't need to build one!

**OpenSofa Web = Tunnel + Session Selector + WhatsApp Bridge**

## 4. MVP Scope: 3 Components Only

### 4.1 Session Launcher Page (Single HTML file)

```html
<!-- Served at tunnel URL root -->
<!DOCTYPE html>
<html>
<head>
  <title>OpenSofa</title>
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <meta name="apple-mobile-web-app-capable" content="yes">
</head>
<body>
  <h1>OpenSofa Sessions</h1>
  <div id="sessions"></div>
  <script>
    // Fetch sessions from /api/sessions
    // Render links to /tunnel/:port/chat
  </script>
</body>
</html>
```

### 4.2 Multi-Port Tunnel

Instead of one tunnel to OpenSofa's web server, we need:

```typescript
// Tunnel each AgentAPI port
const tunnels = {
  3284: 'https://session-a.trycloudflare.com',
  3285: 'https://session-b.trycloudflare.com',
  3286: 'https://session-c.trycloudflare.com',
};
```

**Simpler alternative: Use subpaths**

```
https://tunnel.trycloudflare.com/
  ├── /                    → Session launcher
  ├── /3284/chat           → Proxy to localhost:3284/chat
  ├── /3285/chat           → Proxy to localhost:3285/chat
  └── /api/...             → OpenSofa API
```

### 4.3 WhatsApp Deep Links

When approval needed, WhatsApp sends:

```
⚠️ [frontend] needs approval: npm test

Tap to open: https://tunnel.trycloudflare.com/3284/chat
```

## 6. Tunnel Configuration

### Option A: Single Tunnel with Path Routing (Recommended)

```yaml
# ~/.cloudflared/config.yml
tunnel: <auto-generated>
ingress:
  - hostname: *.trycloudflare.com
    path: /3284/.*
    service: http://localhost:3284
  - hostname: *.trycloudflare.com
    path: /3285/.*
    service: http://localhost:3285
  - hostname: *.trycloudflare.com
    path: /api/.*
    service: http://localhost:3285  # OpenSofa API
  - hostname: *.trycloudflare.com
    path: /.*
    service: http://localhost:3285  # Session launcher
```

### Option B: Quick Tunnel + HTML Links

```bash
# Start quick tunnel
cloudflared tunnel --url http://localhost:3285

# In HTML, link to relative paths:
# <a href="/3284/chat">Open Session A</a>
```

This requires a reverse proxy in OpenSofa to route `/3284/*` to `localhost:3284`.
